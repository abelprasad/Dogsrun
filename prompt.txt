In src/app/dogs/page.tsx, the page is a server component but it's using the browser Supabase client from '@/lib/supabase'. This causes it to only return 1 row due to RLS/auth context issues.

Replace the import and client initialization with the server client:

import { createServerClient } from '@supabase/ssr'
import { cookies } from 'next/headers'

Then inside the function:
const cookieStore = cookies()
const supabase = createServerClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
  {
    cookies: {
      get(name: string) {
        return cookieStore.get(name)?.value
      },
    },
  }
)

Keep everything else exactly the same.
