File: src/app/dashboard/dogs/[id]/edit/edit-form.tsx (or wherever the form submit handler is)

BUG: Saving edits does not persist — the update silently fails.

The issue is the Supabase update is using the anon key client, but RLS on the dogs table 
requires the shelter to own the dog. The client-side session may not be passing through correctly.

Fix the edit form submit by changing it to POST to a new API route instead of calling 
Supabase directly from the client.

Create src/app/api/dogs/update/route.ts:
- Accepts POST with the full dog update payload including dog_id
- Uses Supabase service role key (SUPABASE_SERVICE_ROLE_KEY) to bypass RLS
- Validates that dog_id is present
- Does supabase.from('dogs').update({...fields}).eq('id', dog_id)
- Returns { success: true } or error

In the edit form, replace the direct Supabase update call with:
fetch('/api/dogs/update', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ dog_id, ...formFields })
})

Include all fields: name, breed, mix, age_years, weight_lbs, sex, color, description,
parvo, tripod, blind, other_issues, other_issues_notes
